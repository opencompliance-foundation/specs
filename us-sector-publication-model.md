# U.S. Sector Publication Model

This file defines what OpenCompliance publishes publicly versus privately for the first U.S. sector-regime depth wave.

## HIPAA Privacy Rule

- Source availability: `public_exact_feasible`
- Public allowed:
  - reviewed CFR-section anchors
  - public-control mappings
  - typed claim contracts
  - documentary and rights-workflow boundary notes
- Private allowed:
  - working decomposition notes
  - non-public legal commentary
- Public blocked:
  - claiming that all HIPAA Privacy obligations are mechanised or fully satisfied by the current documentary corridor

## HIPAA Breach Notification Rule

- Source availability: `public_exact_feasible`
- Public allowed:
  - reviewed CFR-section anchors
  - breach-governance and escalation mappings
  - typed runbook and escalation attestations
- Private allowed:
  - incident tabletop notes
  - operator-specific breach playbooks
- Public blocked:
  - claiming that a current runbook proves timely and complete notification performance in every real incident

## GLBA Safeguards Rule

- Source availability: `public_exact_feasible`
- Public allowed:
  - reviewed CFR-section anchors
  - technical subset mappings for MFA, logging, encryption, vendor oversight, incident response, and disposal
- Private allowed:
  - implementation-specific residual-risk notes
  - non-public control-design discussion
- Public blocked:
  - claiming that the current narrow technical overlap equals a full Safeguards Rule program
