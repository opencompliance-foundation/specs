# OpenCompliance Specs

This directory is the private source of truth for the first public export to `opencompliance-foundation/specs`.

The goal is narrow:

- define draft artifact formats,
- keep the formats aligned with the public example fixtures,
- and say clearly what is still missing.

## Current status

These specs are draft `0.x` documents.

They describe the artifact shapes OpenCompliance intends to stabilize:

- mapping methodology and control-boundary metadata,
- exact-anchor review pilots and review-status semantics,
- trust-surface reports,
- proof bundles,
- verification-result envelopes,
- typed punch-lists,
- narrow corridor certificates,
- replay bundles,
- witness receipts,
- revocations,
- public synthetic example packs,
- and versioning rules for those artifacts.

They are useful now for review and example generation. The public synthetic ExampleCo set now spans seven corridor packs, including dedicated cyber-baseline and AI-governance examples. They are not yet a stable standard.
