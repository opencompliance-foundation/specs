# Contributing to OpenCompliance Specs

This repo holds the draft public artifact format documents.

## Ground rules

- Keep specs and public examples aligned.
- Do not call a format stable unless versioning and compatibility expectations say so explicitly.
- Keep normative statements separate from explanatory examples.
- Prefer smaller, reviewable changes over large speculative rewrites.

## Before opening a PR

- Update the affected example bundle if the format shape changes.
- Update conformance vectors when the expected output changes.
- Explain any compatibility break clearly.
- Mark unresolved design choices instead of burying them in vague prose.
